## 第二步：克隆仓库并搭建环境

打开终端（Git Bash 或 PowerShell），执行：

```bash
# 1. 克隆
git clone https://github.com/xiabai2004/quantum-rl-scheduler.git
cd quantum-rl-scheduler

# 2. 一键初始化环境（推荐！）
bash setup.sh                    # Git Bash
# 或
powershell .\setup.ps1           # PowerShell
```

> `setup.sh` / `setup.ps1` 会自动完成：创建虚拟环境 → 安装依赖 → 配置 .env → 创建项目目录 → 验证关键模块。

**手动安装（备选）：**
```bash
python -m venv venv
venv\Scripts\activate            # Windows
pip install -r requirements.txt
cp .env.example .env
```

**安装 Pre-commit（强烈推荐）：**
```bash
pip install pre-commit
pre-commit install               # commit 前自动：Black 格式化 + isort 排序 + flake8 检查
```

**Mock 模式说明：**
- 🟢 Mock 模式（默认，无需改 .env）：不依赖天衍云平台，仿真开发即可
- 🟡 真机模式：去 https://qc.zdxlz.com 注册 → API管理 → 生成 Key → 编辑 .env
- ⚠️ .env 已在 .gitignore 中，不会被推送

**验证环境：**
```bash
python -c "from src.scheduler.env import QuantumSchedulingEnv; print('OK')"
```

## 第五步：日常开发流程

```bash
# 1. 拉最新代码
git checkout main
git pull origin main

# 2. 创建功能分支（以你的任务命名）
git checkout -b feature/parser-optimize

# 3. 写代码...
# ...编辑文件...

# 4. 提交（注意格式！）
git add .
git commit -m "feat: 优化任务解析器性能"

# 5. 推送分支
git push origin feature/parser-optimize
```

### Commit 格式（必须遵守）

```
<type>: <简短描述>

feat: 新功能
fix: 修复 bug
docs: 文档
test: 测试
refactor: 重构
chore: 杂项
```

| 错误 | 正确 |
|------|------|
| `加了新功能` | `feat: 添加任务优先级队列` |
| `fix bug` | `fix: 修复除零错误` |

## 第六步：创建 Pull Request

1. 推送后打开 GitHub 仓库页面，会看到黄色提示条
2. 点击 **Compare & pull request**
3. 填写 PR 描述（关联 Issue 号：`Closes #5`）
4. 点击 **Create pull request**
5. 等队友 Review 通过后合并

**main 分支受保护，必须走 PR 流程，不能直接推送。**

**PR 自动检查（GitHub Actions CI）：**
创建 PR 后，CI 流水线会自动运行，你可以在 PR 页面看到检查状态：
- ✅ `Lint` — 代码格式（Black + isort + flake8）
- ✅ `Test` — 单元测试（Python 3.10/3.11/3.12）
- ✅ `Type Check` — 类型检查（mypy）
- ✅ `PR Automation` — Commit 格式校验

**所有检查必须通过才能合并。** 如果失败，点击检查详情查看错误日志，修复后重新 push 即可。

## 快速命令参考

```bash
# 环境初始化
bash setup.sh                                   # 一键初始化
pre-commit run --all-files                      # 手动触发代码检查

# 代码质量
black src/ scripts/ tests/                      # 格式化代码
isort src/ scripts/ tests/                      # 排序 import
flake8 src/                                     # 代码检查

# 运行测试
pytest tests/ -v                                # 全部测试
python scripts/e2e_test.py                      # 端到端测试

# 快速训练
python scripts/quick_train.py                   # 5000步验证

# 启动 Web 界面
uvicorn src.visualization.app:app --reload --port 8000
```
