## 第二步：克隆仓库并搭建环境

打开终端（Git Bash 或 PowerShell），执行：

```bash
# 1. 克隆
git clone https://github.com/xiabai2004/quantum-rl-scheduler.git
cd quantum-rl-scheduler

# 2. 安装依赖（建议先创建虚拟环境）
python -m venv venv
venv\Scripts\activate        # Windows
pip install -r requirements.txt

# 3. 配置环境变量
cp .env.example .env

# 🟢 Mock 模式（默认，无需改 .env）
#   不依赖天衍云平台，仿真开发即可，90% 的工作不需要真机

# 🟡 真机模式（需要真机调试时）
#   1. 去 https://qc.zdxlz.com 注册账号 → 个人中心 → API管理 → 生成 Key
#   2. 编辑 .env，填入：
#      TIANYAN_API_KEY=你的Key
#      TIANYAN_MOCK_MODE=false
#   3. ⚠️ .env 已在 .gitignore 中，不会被推送，不会像上次一样泄露

# 4. 验证环境
python -c "from src.scheduler.env import QuantumSchedulingEnv; print('OK')"
```

## 第五步：日常开发流程

```bash
# 1. 拉最新代码
git checkout main
git pull origin main

# 2. 创建功能分支（以你的任务命名）
git checkout -b feature/parser-optimize

# 3. 写代码...
# ...编辑文件...

# 4. 提交（注意格式！）
git add .
git commit -m "feat: 优化任务解析器性能"

# 5. 推送分支
git push origin feature/parser-optimize
```

### Commit 格式（必须遵守）

```
<type>: <简短描述>

feat: 新功能
fix: 修复 bug
docs: 文档
test: 测试
refactor: 重构
chore: 杂项
```

| 错误 | 正确 |
|------|------|
| `加了新功能` | `feat: 添加任务优先级队列` |
| `fix bug` | `fix: 修复除零错误` |

---

## 第六步：创建 Pull Request

1. 推送后打开 GitHub 仓库页面，会看到黄色提示条
2. 点击 **Compare & pull request**
3. 填写 PR 描述（关联 Issue 号：`Closes #5`）
4. 点击 **Create pull request**
5. 等队友 Review 通过后合并

**main 分支受保护，必须走 PR 流程，不能直接推送。**

## 注意事项

- 不要修改 `config/config.yaml` 的 `mock_mode: true`
- Python 文件用 4 空格缩进
- 函数/类必须写 docstring
- 不要提交 `.env` 文件（已在 .gitignore 里）
