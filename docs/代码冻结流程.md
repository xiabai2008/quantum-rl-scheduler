## 冻结前检查清单（8/14 完成）

### 1. 拉取最新代码

```bash
git checkout main
git pull origin main
```

### 2. 运行预冻结检查脚本

```bash
bash scripts/ci/pre_freeze_check.sh
```

脚本自动检查 8 项：

| # | 检查项 | 说明 |
|---|--------|------|
| 1 | 代码同步 | 本地与 origin/main 一致 |
| 2 | 分支状态 | 在 main 分支，工作区干净 |
| 3 | CI 状态 | 所有 required checks 通过 |
| 4 | 代码格式 | Black + isort 通过 |
| 5 | 单元测试 | 全部通过，覆盖率 ≥ 60% |
| 6 | 提交物校验 | submission_manifest.yaml 各项齐备 |
| 7 | 数字审计 | 无旧值残留（+95.4%/2864 等） |
| 8 | 文件确认 | README / 模型 / 报告等存在 |

`--quick` 模式跳过慢速检查（CI 状态、全量测试）：

```bash
bash scripts/ci/pre_freeze_check.sh --quick
```

## 冻结后紧急修复流程

冻结后如发现致命 Bug，需遵循以下流程：

1. **创建 hotfix 分支**:
   ```bash
   git checkout -b hotfix/<简短描述> v8.0-submission
   ```

2. **修复 + 提交**:
   ```bash
   git add <修改的文件>
   git commit -m "hotfix: <描述>"
   ```

3. **创建 PR → main**（需要队友 Review）

4. **合并后重新打标签**:
   ```bash
   git checkout main
   git pull origin main
   git tag -a v8.0-submission-hotfix1 -m "v8.0 提交版本 (hotfix 1)"
   git push origin v8.0-submission-hotfix1
   ```

5. **通知团队**：在新标签的 Release 中说明变更内容。

## 常见问题

### Q: 推送标签后 CI 失败怎么办？

查看失败日志 → 修复问题 → 重新打标签（删除旧标签 + 创建新标签）：

```bash
# 删除本地和远程旧标签
git tag -d v8.0-submission
git push origin :refs/tags/v8.0-submission

# 修复代码后重新打标签
git tag -a v8.0-submission -m "v8.0 提交版本"
git push origin v8.0-submission
```

### Q: 冻结日期可以推迟吗？

由瑞哥决策。如推迟，需同步更新 `submission_manifest.yaml` 中的 `deadline` 字段。

### Q: pre_freeze_check.sh 中某些检查失败但不影响提交怎么办？

例如演示视频临时缺失（其他队友负责），可以手动跳过特定检查。但必须确保 `submission_manifest.yaml` 中的所有强制项就位。
