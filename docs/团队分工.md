## 二、10人分工明细

### 👤 角色1：项目经理（1人）— **瑞哥**

| 职责 | 具体内容 |
|------|----------|

---

### 🧠 角色2-4：算法组（3人）

#### 算法工程师 A：RL 智能体开发（核心）
**负责文件：**
- `src/scheduler/agent.py` — DQN 智能体编写与训练
- `src/scheduler/env.py` — Gymnasium 环境优化
- `scripts/train_agent.py` — 训练脚本参数调优

**任务清单：**
- [ ] 实现 DQN/Dueling DQN 网络架构
- [ ] 设计奖励函数（量子加速比 + 等待惩罚 + 利用率奖励）
- [ ] 超参数调优（learning_rate, gamma, epsilon decay）
- [ ] TensorBoard 训练曲线可视化
- [ ] 定期保存模型检查点

#### 算法工程师 B：量子退火加速模块
**负责文件：**
- `src/quantum/annealing.py` — 量子退火优化器
- `scripts/run_simulation.py` — 仿真对比实验

**任务清单：**
- [ ] 实现神经网络权重 → QUBO 矩阵映射
- [ ] 模拟退火求解 QUBO（真机不可用时的仿真方案）
- [ ] D-Wave Ocean SDK 真机 API 对接（拿到权限后）
- [ ] 对比实验：RL only vs RL + 量子退火加速

#### 算法工程师 C：任务解析器 + 系统仿真
**负责文件：**
- `src/scheduler/parser.py` — 任务解析与资源预估
- `src/scheduler/env.py` — 仿真环境（与算法A协作）
- `tests/test_scheduler.py` — 单元测试

**任务清单：**
- [ ] 实现 TaskParser 完整功能（解析、校验、资源预估）
- [ ] 任务生成器（四种负载模式：均匀、突发、周期、混合）
- [ ] 编写仿真测试用例（覆盖边界情况）
- [ ] 性能基准测试（FCFS / 随机 / 贪心 作为 baseline）

---

### ⚙️ 角色5-7：工程组（3人）

#### 后端工程师 A：天衍云 API 集成
**负责文件：**
- `src/api/tianyan_client.py` — API 封装
- `config/config.yaml` — 配置管理

**任务清单：**
- [ ] 实现天衍云 REST API 完整封装
- [ ] 任务提交流程（QASM → 提交 → 轮询 → 结果）
- [ ] 错误处理与指数退避重试
- [ ] 多 backend 支持（天衍-287超导 / 九章四号光量子）
- [ ] API 响应 Mock（方便算法组无真机时开发）

#### 后端工程师 B：调度核心 + 数据库
**负责文件：**
- `src/scheduler/` — 调度引擎（与算法组协作）
- `src/utils/helpers.py` — 工具函数

**任务清单：**
- [ ] 调度器核心逻辑（接收任务 → 调用RL → 分发到经典/量子）
- [ ] SQLite/JSON 任务日志存储
- [ ] 系统状态监控（CPU/内存/API调用量）
- [ ] WebSocket 推送（供前端实时展示）

#### 测试工程师：系统测试 + CI/CD
**负责文件：**
- `tests/` — 测试用例
- `.github/workflows/` — GitHub Actions

**任务清单：**
- [ ] 单元测试覆盖（pytest，目标 >80%）
- [ ] 集成测试（API Mock + 仿真环境）
- [ ] GitHub Actions CI（自动运行测试、代码格式检查）
- [ ] 性能压测（1000+任务并发调度）
