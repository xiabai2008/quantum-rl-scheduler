> 适用对象：第一次把本项目连上天衍云真机、跑 PPO 真机闭环训练的队友。
> 关联代码：`scripts/training/train_agent_real.py`、`src/api/tianyan_cqlib.py`、`src/scheduler/env_real_machine.py`

## 0. 一句话流程

```
拿到 API Key  →  装 cqlib  →  配 .env  →  (可选)验证连接  →  跑 train_agent_real.py
```

没有 API Key 时，脚本会**自动退化为纯仿真训练**，不报错，可正常跑通流程；但要真正用真机，必须完成前三步。

## 2. 安装真机依赖

基础依赖照常安装，再单独装机机 SDK：

```bash
# 1) 基础依赖（含 python-dotenv，已在 requirements.txt）
pip install -r requirements.txt

# 2) 真机 SDK cqlib（从天衍云内部源）
pip install -r requirements-quantum.txt
# 或等价手动指定源：
# pip install cqlib -i https://pypi.tianyan.com/simple/
```

验证 cqlib 是否装好：

```bash
python -c "import cqlib; print('cqlib', cqlib.__version__)"
```

> 若提示找不到 `cqlib` 或版本不兼容：确认 Python 版本在 cqlib 支持的区间内；必要时从天衍云平台下载对应 `.whl` 手动 `pip install`。

## 3. 配置 `.env`

项目根目录已有 `.env.example`，复制为 `.env` 并填入真实值：

```bash
cp .env.example .env
```

修改以下三项（其余保持默认即可）：

```ini
# 真实模式必需：填天衍云 API Key（TIANYAN_API_TOKEN 等效，二选一）
TIANYAN_API_KEY=你拿到的真实API密钥

# 默认量子计算机（可选值见下方机器列表）
TIANYAN_MACHINE=tianyan_s

# 关闭 Mock 模式，走真实平台
TIANYAN_MOCK_MODE=false
```

> ⚠️ **安全提醒**：`.env` 已在 `.gitignore` 中，提交代码时绝不会带上真实 Key。
> 如果误把 Key 提交，立即在平台吊销并重新生成。

可选机器名（`src/api/tianyan_cqlib.py` 中 `REAL_MACHINES`）：
`tianyan_sw`、`tianyan_s`、`tianyan_tn`、`tianyan_tnn`、`tianyan_swn`、`tianyan_sa`、`tianyan176`、`tianyan176-2`。
本项目真机训练脚本默认纳管 `DEFAULT_MACHINE_CONFIGS` 中的 **tianyan_s / tianyan_sw / tianyan_tn** 三台。

## 4. 验证真机连接（强烈建议先做）

在正式跑训练前，用几行代码确认 Key 有效、机器在线：

```python
# 在项目根目录下执行
from src.api.tianyan_cqlib import CqlibTianyanClient

client = CqlibTianyanClient(login_key="你的Key", machine_name="tianyan_s")

print("认证是否通过:", client.authenticate())        # True 表示 Key 有效
print("机器列表:", client.list_backends())            # 列出可用超导真机及状态
```

- `authenticate()` 返回 `True` → Key 正确，可继续。
- 返回 `False` 或抛错 → 检查 Key 是否复制完整、是否过期、网络是否可连平台。
- 机器 `status` 必须是 `running` 才能提交；`calibration`/`maintenance` 表示校准/维护中，会触发自动切换或降级。

## 5. 进入真机训练

核心入口脚本：`scripts/training/train_agent_real.py`

### 5.1 最常用命令（省机时、先跑通）

```bash
# 从项目根目录执行
python scripts/training/train_agent_real.py \
    --timesteps 5000 \
    --real-prob 0.05
```

这条命令会：
1. **先跑纯仿真训练**（基线，label=sim）
2. **再跑真机混合训练**（label=real，按 5% 概率把量子任务上真机）
3. 评估两个模型并生成对比报告 `results/reports/real_machine_training_comparison.md`

### 5.2 纯仿真（无 Key 或不想用真机）

```bash
python scripts/training/train_agent_real.py --timesteps 5000 --episodes 5
```
没有 `TIANYAN_API_KEY` 时自动退化为此模式，仅验证代码链路。

### 5.3 关键参数说明

| 参数 | 默认 | 含义 | 建议 |
|------|------|------|------|
| `--timesteps` | 5000 | 每种模式训练总步数 | 先 5000 跑通，再上 1e4~1e5 |
| `--real-prob` | 0.05 | 每个量子任务上真机的概率 | **控制在 0.01–0.05**，省机时 |
| `--real-callback-interval` | 1000 | 训练回调中触发真机提交的间隔步数 | 默认即可 |
| `--episodes` | 5 | 评估 episode 数 | 5 足够 |
| `--seed` | 42 | 随机种子 | 与实验报告保持一致 |
| `--save-dir` | models/ | 模型保存目录 | 默认 |
| `--output` | results/reports/real_machine_training_comparison.md | 对比报告路径 | 默认 |

### 5.4 只生成对比报告（已有两个模型时）

```bash
python scripts/training/train_agent_real.py \
    --report-only \
    --sim-model models/ppo_sim_seed_42.zip \
    --real-model models/ppo_real_seed_42.zip
```

## 6. 机时控制与成本意识（重要）

真机机时是团队申请的有限资源，请务必省着用：

- **`--real-prob` 设小**：0.01–0.05 足够验证闭环，不要用 1.0。
- **shots 默认 512**：代码内部提交默认 512 次测量，已是合理值，勿盲目调大。
- **默认不传 QuotaTracker**：脚本默认不做配额硬限制，机时完全由你手动控制，请自觉。
- 真机任务提交是非阻塞的：提交后立刻返回，结果在后续 `step()` 轮询，不会卡住训练。

## 7. 自动降级机制（正常现象，不是 bug）

`src/scheduler/env_real_machine.py` 内置保护：

- 真机**连续失败达到 3 次**（`REAL_MACHINE_DEGRADE_FAIL_THRESHOLD`），自动降级到 Mock，不再消耗机时。
- 触发后会打印：`[真机闭环] 已降级到 Mock（连续失败 3 次）`。
- 常见导致失败的原因：机器校准/维护中、容量超限、网络抖动。
- **降级后训练不中断**，只是剩余部分走仿真。想恢复真机，重新运行脚本即可（前提是机器恢复 running）。

---

## 8. 免费机时包的限制（必读，否则容易踩坑）

`env_real_machine.py` 中的实测结论：

- **免费机时包仅 1-qubit 电路稳定**（`FREE_TIER_MAX_QUBITS = 1`）。
- 代码默认只生成**单比特门电路**（`two_qubit_gates=False`），确保高成功率。
- 两比特门（CNOT/CZ）在免费真机上不稳定，Bell 态有失败率，**默认不启用**。
- 若电路比特数超出机时包额度，会报「您的机时包最大比特数不支持本任务」——这是容量错误，代码会自动跳过该任务，不会崩。

> 结论：队友无需关心电路生成细节，脚本已处理。只需知道「真机任务都是小规模单比特电路，这是平台限制，不是我们代码的局限」。

## 9. 常见问题排查

| 现象 | 原因 | 处理 |
|------|------|------|
| 提示「未配置 TIANYAN_API_KEY，将退化为纯仿真」 | 没填 Key / `.env` 没加载 | 确认 `.env` 存在且 `TIANYAN_API_KEY` 已填；脚本用 `load_dotenv()` 读取 |
| `authenticate()` 返回 False | Key 无效/过期/网络不通 | 重领 Key；换网络；确认能连 `pypi.tianyan.com` 和平台 API |
| 提交后立刻降级 | 机器校准/维护中 | 换 `TIANYAN_MACHINE` 或等平台恢复 running |
| 「机时包最大比特数不支持」 | 电路超免费额度 | 代码已自动跳过，无需处理；属平台限制 |
| 真机提交数为 0 | `real-prob=0` 或未启用真机 | 确认 `--real-prob>0` 且 `TIANYAN_API_KEY` 有效 |
| cqlib 安装失败 | Python 版本不支持 / 源不可达 | 确认 Python 3.10+；检查内部源可达；手动下载 wheel |
