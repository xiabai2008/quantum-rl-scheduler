## 6. 结论与建议

### 6.1 真机可用性评估

- **单比特门（H门）**: 保真度稳定在 0.97-0.99，性能可靠
- **双比特门（CZ门）**: Bell 态实验失败，当前硬件双比特门不稳定
- **任务提交延迟**: 平均 0.5-0.8s/任务，可接受
- **任务执行**: 部分任务因 SDK 内部重试超时失败（~20% 失败率）

### 6.2 性能瓶颈分析

- **SDK 超时**: cqlib SDK 在任务失败时进入无限内部重试，需 daemon 线程超时机制兜底
- **双比特门**: 硬件层面 CZ 门保真度不足，限制多比特电路验证
- **机时限制**: 限免窗口期机时有限，需控制 shots 和任务数

### 6.3 后续优化方向

1. **PPO 真机闭环**: 将 PPO 策略推理结果直接注入真机调度循环
2. **多机器协调**: 利用 `tianyan176-2` / `tianyan_sw` / `tianyan_s` 备用机分流
3. **QAOA 验证**: 在硬件改善后尝试 QAOA 算法验证 QUBO 求解
4. **DQN 模型重训**: 使用 14 维观测空间重新训练 DQN 模型
5. **双比特门重试**: 增加 CZ 门电路的自动重试和错误恢复机制
