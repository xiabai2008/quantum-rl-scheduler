# 真机闭环实验预注册方案（Pre-Registration）

> **Issue**: #221 — 真机闭环预注册验证：以效应量/CI为主，禁止追p值
> **生成时间**: 2026-07-21
> **预注册日期**: 2026-07-21（数据收集前）
> **适用范围**: 所有真机闭环性能对比实验
> **核心原则**: 效应量（Cohen's d）+ 95% 置信区间（CI）为主要决策指标；p 值为辅助参考，禁止作为唯一决策依据

---

## 一、预注册声明

### 1.1 为什么需要预注册

真机实验面临样本量小（N=5-10/组）、硬件资源受限（免费机时包）、数据变异大（真机噪声）等固有约束。在样本量不足时，p 值极不稳定（同一效应量在 N=5 vs N=50 下 p 值可相差数个数量级），容易导致以下问题：

- **p-hacking**：通过调整分析参数（检验方法、异常值剔除标准、子集选择）使 p 值跨过 0.05 阈值
- **选择性报告**：只报告 p<0.05 的对比，隐藏不显著的结果
- **事后假设**：看到数据后修改假设，制造"显著"结论

预注册的核心目的是**在数据收集前固定分析方案**，消除分析者自由度，确保结论的可信度。

### 1.2 本预注册的约束力

- 本文档在真机实验数据收集**之前**编写并提交至仓库
- 后续真机实验分析**必须**遵循本文档定义的分析计划
- 如需偏离预注册方案，必须在报告中**显式声明**偏离原因及影响
- 任何未在预注册中定义的事后分析（exploratory analysis）必须标注为"探索性分析"，不作为正式结论

## 二、研究问题与假设

### 2.1 研究问题

**RQ**: 在天衍云真机闭环训练条件下，真机测量结果反馈（`result_aware` 模式）是否对 PPO 调度策略性能产生实质性影响？

### 2.2 预注册假设

| 假设编号 | 类型 | 描述 | 方向 |
|:--------:|:----:|:-----|:----:|
| H1 | 主假设 | `result_aware`（保真度感知奖励）模式的平均奖励**高于** `status_only`（固定奖励）模式 | 单侧 |
| H2 | 主假设 | `result_aware` 模式的平均奖励**高于** `shuffled`（打乱结果消融对照）模式 | 单侧 |
| H3 | 次要假设 | `status_only` 模式与 `shuffled` 模式之间**无实质差异**（|d| < 0.2） | 双侧 |

### 2.3 假设的科学依据

- **H1 依据**：`result_aware` 模式将真机测量结果的保真度（fidelity）编码进 RL 奖励信号，理论上应引导 PPO 学习到更精准的量子任务调度策略。如果真机反馈有信息价值，则奖励信号应优于固定奖励。
- **H2 依据**：`shuffled` 模式打乱真机结果与任务的对应关系，破坏了保真度信息但保留了结果的统计分布。如果 H2 成立，说明 PPO 确实利用了保真度信息而非仅仅依赖结果的存在性。
- **H3 依据**：`status_only` 和 `shuffled` 都不提供有效的保真度信息（前者无结果，后者结果错配），两者性能应无实质差异。

## 四、主要评价指标

### 4.1 主要指标：效应量 + 置信区间

> **核心原则**：以效应量（Cohen's d）和 95% Bootstrap CI 为主要决策依据，不以 p 值作为唯一标准。

| 指标 | 计算方法 | 实现函数 | 决策阈值 |
|:-----|:---------|:---------|:---------|
| **Cohen's d** | d = (mean_A - mean_B) / pooled_std | `src/utils/stats_significance.py::cohen_d()` | 见 §4.3 |
| **95% Bootstrap CI（提升百分比）** | 10000 次重抽样，百分位法 | `bootstrap_improvement_ci()` | CI 下界 > 0 |
| **均值差 95% CI** | t 分布 CI | `_mean_diff_ci()` | CI 不跨 0 |

### 4.2 辅助指标（参考性质）

| 指标 | 计算方法 | 角色 |
|:-----|:---------|:-----|
| p 值 | 自动选择检验（t/Welch t/Mann-Whitney U） | **辅助参考**，不作为主要决策依据 |
| rank-biserial correlation | 非参数效应量 | 非正态数据的补充效应量 |
| 统计功效（post-hoc） | 基于观测效应量计算 | 评估结论可靠性 |
| Bonferroni 校正 α | α / 比较次数 | 多重比较误差控制 |

### 4.3 效应量决策矩阵

| Cohen's d | 等级 | 解读 | 决策 |
|:---------:|:----:|:-----|:-----|
| \|d\| < 0.2 | 可忽略 | 真机反馈无实质影响 | **H1/H2 不支持** |
| 0.2 ≤ \|d\| < 0.5 | 小效应 | 真机反馈有微弱影响 | **弱支持**（需更多数据） |
| 0.5 ≤ \|d\| < 0.8 | 中效应 | 真机反馈有中等影响 | **支持**（需 CI 验证） |
| \|d\| ≥ 0.8 | 大效应 | 真机反馈有显著影响 | **强支持** |

> **重要**：效应量等级是必要条件但非充分条件。最终结论需结合 95% CI 判断：
> - **CI 下界 > 0**：效应方向可信
> - **CI 下界 ≤ 0**：效应方向不可信，即使点估计为正

## 五、样本量与停止规则

### 5.1 功效分析

基于现有真机实验数据（Issue #164）的初步估计：

| 参数 | 值 | 来源 |
|:-----|:---|:-----|
| 观测均值差 | 1025.51 | `real_machine_statistical_significance.md` |
| 合并标准差 | 1511.22 | 同上 |
| 观测 Cohen's d | 0.6786（中效应） | 同上 |
| 目标功效 | 80% | 统计惯例 |
| 显著性水平 α | 0.05（Bonferroni 校正后 0.017） | 3 组两两比较 = 3 次 |
| **所需样本量** | **≥ 18 组/条件** | G*Power 计算 |

### 5.2 样本量计划

| 阶段 | 样本量/组 | 目的 | 停止条件 |
|:-----|:----------|:-----|:---------|
| 第一阶段 | 18 seeds × 5 episodes = 90 | 预注册主分析 | 达到最低功效要求 |
| 第二阶段（如需要） | 追加至 30 seeds × 5 = 150 | 提升效应量估计精度 | CI 宽度 < 300 |

### 5.3 停止规则

1. **最低样本量**：每组不少于 18 seeds（90 个数据点），低于此数量不进行假设检验
2. **最高样本量**：每组不超过 30 seeds（150 个数据点），避免过度收集
3. **中间停止**：如真机平台不可用（连续 3 天无法提交任务），停止收集并报告已有数据
4. **异常停止**：如某条件连续失败率 > 20%，停止该条件并报告原因

### 5.4 数据排除标准（预定义）

以下数据**在分析前**将被排除，排除决策不依赖于分析结果：

| 排除条件 | 判定标准 | 处理方式 |
|:---------|:---------|:---------|
| 真机调用失败 | 真机成功率 < 80% 的 seed | 排除该 seed 全部数据 |
| Episode 未完成 | episode 步数 < 200 且未正常终止 | 排除该 episode |
| 异常值 | 奖励超出 [mean ± 4σ]（**4 倍标准差**，非 2 倍） | 排除该数据点，记录排除原因 |
| Mock 降级 | 连续 3 次真机失败触发 Mock 降级 | 排除降级后的数据，保留降级前数据 |

> **关键**：异常值剔除标准使用 4σ 而非常见的 2σ 或 3σ，以最大程度减少分析者自由度。所有排除必须记录在数据审计日志中。

## 六、分析计划

### 6.1 分析流程

```
数据收集完成
    │
    ▼
数据清洗（按 §5.4 预定义标准排除）
    │
    ▼
描述性统计（均值、标准差、中位数、IQR）
    │
    ▼
正态性检验（Shapiro-Wilk / D'Agostino K²）
    │
    ├── 正态 → Cohen's d + t 分布 CI
    │
    └── 非正态 → rank-biserial + Bootstrap CI
    │
    ▼
假设检验（辅助参考）
    │
    ▼
效应量 + CI 决策（主要依据）
    │
    ▼
结论判定
```

### 6.2 两两比较计划

| 比较编号 | 比较 | 对应假设 | 主指标 |
|:--------:|:-----|:---------|:-------|
| Comp-1 | C2 (`result_aware`) vs C1 (`status_only`) | H1 | Cohen's d + 95% CI |
| Comp-2 | C2 (`result_aware`) vs C3 (`shuffled`) | H2 | Cohen's d + 95% CI |
| Comp-3 | C1 (`status_only`) vs C3 (`shuffled`) | H3 | Cohen's d + 95% CI |

多重比较校正：Bonferroni 校正，α_adj = 0.05 / 3 = 0.0167

### 6.3 分析代码

使用项目现有的统计模块，确保分析可复现：

```python
from src.utils.stats_significance import (
    compare_strategies,
    cohen_d,
    rank_biserial,
    bootstrap_improvement_ci,
    normality_test,
)

# 1. 描述性统计
# 2. 正态性检验
# 3. compare_strategies() 自动选择检验方法并计算效应量
# 4. bootstrap_improvement_ci() 计算提升百分比 CI
```

### 6.4 结论判定标准

| 假设 | 判定为"支持"的条件 | 判定为"不支持"的条件 |
|:-----|:-------------------|:---------------------|
| H1 | d ≥ 0.5 **且** CI 下界 > 0 | d < 0.2 **或** CI 跨 0 |
| H2 | d ≥ 0.5 **且** CI 下界 > 0 | d < 0.2 **或** CI 跨 0 |
| H3 | \|d\| < 0.2 | \|d\| ≥ 0.5 |

> **中间区域**（0.2 ≤ d < 0.5 或 CI 跨 0）：判定为"不确定，需更多数据"，**不**强行得出结论。

## 七、禁止的 p 值操纵行为

以下行为在本预注册中**明确禁止**：

### 7.1 禁止行为清单

| 编号 | 禁止行为 | 说明 |
|:----:|:---------|:-----|
| P1 | **选择性报告** | 只报告 p<0.05 的比较，隐藏不显著结果 |
| P2 | **事后调整检验方法** | 看到数据后切换 t 检验/Welch t/Mann-Whitney 以获得更小的 p 值 |
| P3 | **事后调整异常值标准** | 通过调整剔除标准使 p 值跨过 0.05 |
| P4 | **事后增加样本量** | 看到 p=0.06 后继续收集数据直到 p<0.05（optional stopping） |
| P5 | **事后选择子集** | 只分析表现好的 seeds 或特定 episode 范围 |
| P6 | **多重比较不校正** | 进行多次比较但不报告 Bonferroni 校正 |
| P7 | **p 值四舍五入** | 将 p=0.054 报告为 p<0.05 或 p=0.05 |
| P8 | **事后修改假设方向** | 看到数据后从单侧检验改为双侧检验（或反之） |

### 7.2 允许的探索性分析

以下分析允许，但**必须标注为"探索性分析"**，不作为正式结论：

- 不同 episode 长度下的性能趋势分析
- 真机调用次数与奖励的散点图回归
- 按任务类型（量子/经典/混合）分层的子组分析
- 按机器类型分层的子组分析

## 九、报告模板

实验完成后的报告**必须**包含以下章节：

### 9.1 必报章节

1. **实验概述**：条件、参数、实际样本量
2. **数据清洗报告**：排除的数据点及原因（按 §5.4 标准）
3. **描述性统计**：每组的均值、标准差、中位数、IQR
4. **正态性检验**：每组的检验方法和 p 值
5. **效应量报告**（主要）：
   - Cohen's d（或 rank-biserial）
   - 95% CI（均值差 CI + Bootstrap 提升百分比 CI）
   - 效应量等级判定
6. **假设检验报告**（辅助）：
   - 检验方法、统计量、p 值
   - Bonferroni 校正后的 α
7. **假设判定**：每个假设的"支持/不支持/不确定"判定
8. **功效分析**：post-hoc 功效，是否达到 80%
9. **探索性分析**（如有）：明确标注为探索性
10. **偏离声明**（如有）：任何偏离预注册方案的行为及原因

### 9.2 报告标准

- 所有效应量报告至小数点后 4 位
- 所有 p 值报告精确值（不使用 p<0.05 等近似），极小值可使用科学计数法
- 所有 CI 报告 [下界, 上界] 格式
- 禁止使用"趋势性显著"、"边缘显著"等模糊表述

## 十二、总结

本预注册方案的核心设计原则：

1. **效应量优先**：Cohen's d + 95% CI 是主要决策依据，p 值为辅助参考
2. **预定义分析**：所有分析参数（检验方法、异常值标准、样本量）在数据收集前固定
3. **禁止 p-hacking**：8 类禁止行为明确列出
4. **透明报告**：所有偏离预注册的行为必须显式声明
5. **诚实结论**：允许"不确定"作为正式结论，不强行得出二元判定

> **核心信条**：在小样本真机实验中，诚实的"不确定"比虚假的"显著"更有科学价值。
